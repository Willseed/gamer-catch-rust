GamerCatch 使用指南

不會寫程式，也能完成每日排行抓取

從下載、Google Sheets、Gmail 通知到 Windows 每日排程，照順序操作即可。 同一份設定可以處理多個遊戲，也能連到不同人的 Google 試算表。

這個網站不收憑證。 請勿上傳 service account JSON、Gmail OAuth JSON、private key、client secret 或 refresh token。設定產生器只需要檔名與相對路徑。

01

下載正確版本並完整解壓縮

最常見的失敗原因，就是直接在 ZIP 裡執行，或只把其中一個檔案拖出來。 請先確認電腦類型，再下載對應版本。

macOS

Apple Silicon

  1. 點左上角蘋果選單，再點「關於這台 Mac」。本版支援 Apple M1、M2、M3、M4 等 Apple Silicon；Intel Mac 暫不支援。
  2. 下載 GamerCatch-macOS-arm64.zip。
  3. 雙擊 ZIP，把完整的 GamerCatch 資料夾移到「文件」等固定位置。
  4. 確認資料夾內有 GamerCatch、三個 .command、 config.toml、credentials 與 playwright-driver。

Windows

Intel / AMD x64

  1. 開啟「設定」→「系統」→「關於」。本版支援一般 Intel 或 AMD 的 x64 電腦；Windows on ARM 暫不支援。
  2. 下載 GamerCatch-Windows-x64.zip。
  3. 在 ZIP 上按右鍵，選「全部解壓縮」，再進入新資料夾操作。
  4. 確認資料夾內有 GamerCatch.exe、四個 .cmd、 install-windows-task.ps1、config.toml、 credentials 與 playwright-driver。
不要拆散資料夾 主程式、雙擊腳本與 playwright-driver 必須放在一起。也不要直接從壓縮檔預覽視窗執行。
02

第一次雙擊與系統安全提示

只從官方 GitHub Release 下載 先核對 repository 是 Willseed/gamer-catch-rust，並依 SHA-256 章節核對檔案。不要關閉 Gatekeeper、SmartScreen 或防毒軟體。

第一次設定

  1. macOS 雙擊 1_首次設定.command；Windows 雙擊 1_首次設定.cmd。第一次會準備 Chromium，可能需要數分鐘。
  2. 保持網路連線與黑色視窗開啟，直到顯示完成。
  3. 完成後會協助您找到 config.toml、credentials 資料夾與這份線上指南。

macOS

Gatekeeper 阻擋時

目前預覽版尚未使用 Developer ID 簽章與 Apple 公證。核對來源後，按住 Control 點擊腳本並選「打開」；若仍被阻擋，到「系統設定」→「隱私權與安全性」選「仍要打開」。

不要執行來路不明的 xattr、spctl 指令，也不要停用 Gatekeeper。

Windows

SmartScreen 警告時

目前預覽版尚未簽章，可能顯示「未知的發行者」。只有檔案來源與雜湊都正確時，才考慮「其他資訊」→「仍要執行」。

公司電腦若禁止執行，請交由 IT 管理員確認，不要停用 Windows 安全性。

03

開始前：整理多遊戲與多人帳號

每個遊戲可以屬於不同的人、不同 Google 帳號、不同 Cloud 專案、不同 JSON 與不同試算表，完全不需要共用帳號。先填一個也可以，其餘遊戲暫時停用。

編號 遊戲名稱 試算表擁有者 分頁 JSON 檔名 欄位
1 夜鴉 第一位使用者 每日排名 person-1-service-account.json A / B / C
2 第二個完整名稱 第二位使用者 每日排名 person-2-service-account.json A / B / C
3 第三個完整名稱 第三位使用者 每日排名 person-3-service-account.json A / B / C

每個遊戲要先準備五樣資料

  • 巴哈排行卡片顯示的完整遊戲名稱，包含空格、全形符號與系列名稱。
  • Google 試算表完整網址。
  • 試算表底部的工作表分頁名稱，例如「每日排名」。
  • 日期、排行、人氣所在欄位，例如 A、B、C。
  • 該遊戲使用的 service account JSON；三個遊戲可以使用三份不同 JSON。
還沒有 JSON 沒關係 先記住「每個遊戲都要指向自己使用的 JSON」。接著依 第 5 章啟用 Sheets API，再到 第 6 章建立並下載 service account JSON；不需要現在就會。
同一款遊戲也能交給不同人 建立兩個相同 game_name 的遊戲項目，分別連到不同試算表、JSON 與通知收件人即可。程式會共用抓到的結果，再分別寫入。
04

準備每一張 Google 試算表

程式只尋找已經存在的今天，不會自行新增日期列。建議每張試算表先建立以下欄位。

A 欄 B 欄 C 欄
日期 排行 人氣
2026/7/21 由程式寫入 由程式寫入
2026/7/22 由程式寫入 由程式寫入
  1. 在 A2 之後輸入包含年份的真正日期，例如 2026/7/21。不要加單引號，也不要只放沒有年份的純文字。
  2. 確認底部分頁名稱；之後的 worksheet_name 必須逐字相同。
  3. 今天只能出現一次，否則程式無法判斷應寫入哪一列。
  4. 執行期間不要排序、插入或刪除資料列。
05

啟用 Google Sheets API

每位試算表使用者都可以用自己的 Google 帳號與 Cloud 專案完成，不需共用同一個管理者帳號。

  1. 使用該張試算表管理者的 Google 帳號登入 Google Cloud Console。
  2. 從上方專案選單建立或選擇專案，例如「GamerCatch-夜鴉」。
  3. 左上角選單 →「API 和服務」→「程式庫」。
  4. 搜尋「Google Sheets API」，進入結果後按「啟用」。建立 API key 並不等於完成。
Sheets 與 Gmail 是兩套不同憑證 Sheets 使用 service account；Gmail 使用寄件者本人同意的 Desktop OAuth。兩份 JSON 不能互相替代。
06

建立 Sheets service account 並下載 JSON

  1. Google Cloud 左上角選單 →「IAM 與管理」→「服務帳戶」→「建立服務帳戶」。
  2. 取一個容易辨認的名稱，例如 gamercatch-nightcrow。
  3. 不需要授予 Cloud 專案 Owner 或 Editor；若畫面允許，略過角色直接完成。試算表權限會在下一章單獨分享。
  4. 點進新帳戶 →「金鑰」→「新增金鑰」→「建立新的金鑰」→選 JSON →「建立」。瀏覽器會下載一份 .json。
JSON 就是程式專用密碼 私鑰無法再次下載。不要上傳 GitHub、雲端公開連結、群組聊天室或問題回報附件。若外洩，立刻到 Google Cloud 刪除舊金鑰並建立新的。
公司或學校禁止建立金鑰？ 這通常是組織安全政策，不是程式故障。請聯絡 Google 管理員，不要嘗試繞過政策。
07

把試算表分享給正確的 service account

您本人能開啟試算表，不代表程式有權限。service account 是另一個機器帳號，必須單獨分享。

  1. 用 TextEdit 或記事本在自己的電腦開啟 JSON，尋找 client_email。
  2. 只複製引號內的電子郵件；不要把整份 JSON 貼到網站或聊天室。
  3. 開啟正確的 Google Sheet，按「共用」，貼上地址並選「編輯者」。service account 沒有收件匣，可以取消寄送通知。
  4. 逐一核對：第一張表對第一份 JSON，第二張表對第二份 JSON，依此類推。
"client_email": "gamercatch-nightcrow@example-project.iam.gserviceaccount.com"
只分享特定試算表即可 不需要把整個 Google Drive 公開，也不需要把 service account 設成網域管理員。
08

把 JSON 放進 credentials 資料夾

  1. 建議依序改名為 person-1-service-account.json、 person-2-service-account.json、 person-3-service-account.json。
  2. 把檔案拖進 GamerCatch 的 credentials 資料夾。
  3. Windows 建議開啟「檢視」→「顯示」→「副檔名」，避免實際檔名變成 .json.json。 
GamerCatch 資料夾/
  config.toml
  credentials/
    person-1-service-account.json
    person-2-service-account.json
    person-3-service-account.json
    gmail-oauth-client.json    # 啟用 Gmail 時才需要
不要把自己的檔案重新壓回公開 ZIP 官方下載包的 credentials 是空資料夾。您的 JSON 只應留在自己的電腦，也絕對不要上傳到本網站。 Gmail refresh token 會存入系統安全儲存區，不會放在這個資料夾。
09

Gmail：建立專用 Desktop OAuth 憑證

Gmail 通知是選用功能。只有希望在排行或人氣抓取異常時收到信，才需要完成本章；不用時保持 gmail_notifications.enabled = false。

不能沿用 Sheets 的 JSON Gmail 使用寄件者本人同意的 OAuth 2.0；Sheets service account、API key 或 Web client 都不能代替 Desktop app OAuth。
  1. 以預定的寄件 Gmail 登入 Google Cloud Console，建立或選擇專案；到「API 和服務」→「程式庫」啟用 Gmail API。
  2. 設定 Google Auth Platform 的應用程式名稱與支援信箱。Workspace 組織內可依政策選 Internal；一般個人 Gmail 選 External，Testing 時把寄件帳號加入 Test users。
  3. 到 Data Access 只加入 https://www.googleapis.com/auth/gmail.send，不要加入讀信或完整信箱權限。
  4. 到 Clients 建立 OAuth client，Application type 必須是 Desktop app（電腦版應用程式）。下載後改名為 gmail-oauth-client.json。
External Testing 通常只有 7 天 gmail.send 授權與 refresh token 通常會在 7 天後失效，不適合長期每日排程。長期使用請依帳號情況採 Internal，或切換 In production 並完成 Google 要求的設定或驗證。
10

Gmail：填收件人並完成首次授權

一份設定支援一個寄件 Gmail 與多位收件人。授權只在第一次、權限失效、換寄件人或換系統使用者時操作；平常抓取不會開 Google 登入頁。 

[gmail_notifications]
enabled = true
sender_email = "alerts@gmail.com"
default_recipients = ["owner@example.com", "backup@example.com"]
oauth_client_secret_path = "credentials/gmail-oauth-client.json"
subject_prefix = "[GamerCatch]"

[[games]]
notification_recipients = []

[[games]]
notification_recipients = ["person-2@example.com"]
  1. 把 OAuth JSON 放入 credentials，並在設定產生器填入它的相對路徑。
  2. 核對 sender_email 和稍後登入的 Gmail 相同。遊戲收件人留空時使用全域預設；填入時會取代全域預設。
  3. macOS 雙擊 Gmail_首次授權.command；Windows 雙擊 Gmail_首次授權.cmd。登入寄件帳號並同意 gmail.send，5 分鐘內完成。
  4. 確認每位有效收件人都收到測試信。結果會寫入 last-gmail-authorization.log。
每個電腦登入帳號都要各自授權 refresh token 只存於目前使用者的 macOS Keychain 或 Windows Credential Manager，不會跟著 ZIP 或 config.toml 複製。換電腦、換使用者、換寄件人、撤銷權限或授權到期，都要重新執行首次授權。
11

Gmail：通知範圍與常見錯誤

每位收件人在一次執行中最多收到一封彙整信，而且只包含他負責的遊戲。

情況 收件規則
遊戲收件人為空 使用全域 default_recipients。
遊戲有專用地址 取代全域地址，只通知該遊戲的 notification_recipients。
同一人負責多個遊戲 地址會去重，失敗項目合併成一封。
全部遊戲的共同錯誤 通知所有啟用遊戲的有效收件人。

什麼情況會寄信？

  • 會寄：瀏覽器、網路、HTTP、安全驗證、DOM、排行或人氣解析、找不到遊戲，以及已啟用的 Sheets 寫入異常。
  • 不會寄：全部成功、強制 dry-run，或刻意設定不寫入 Google Sheets。
  • 設定檔壞掉、程式根本沒啟動、整台電腦斷網或 Gmail 本身失敗時，信可能寄不出去；仍要查看執行 log。
  • 寄信錯誤不會蓋掉原始抓取或寫入錯誤。
看到的問題 處理方式
redirect_uri_mismatch 重新下載 Desktop app OAuth JSON；Web client 或 service account 都不適用。
access_denied External Testing 將寄件帳號加入 Test users；組織政策封鎖時聯絡管理員。
找不到授權 用相同作業系統帳號重新執行 Gmail 首次授權。
本機回呼逾時 保持黑色視窗開啟，並在 5 分鐘內完成瀏覽器授權。
Gmail API 403 確認 API 已啟用且只有 gmail.send；權限更改後撤銷並重新授權。
12

使用網頁設定產生器

不需要自己排 TOML 格式。產生器會在瀏覽器內整理欄位，下載可直接使用的 config.toml。

開啟設定產生器
  1. 先選擇巴哈掃描頁數與是否顯示瀏覽器。一般使用者保留預設即可。
  2. 按「新增遊戲」，逐一填遊戲名稱、試算表網址、分頁、JSON 相對路徑、日期列與輸出欄位。
  3. 若要通知，再開啟 Gmail，填寄件地址、收件地址與 OAuth JSON 相對路徑。
  4. 先讓所有遊戲保持「不寫入 Google Sheets」，再下載 config.toml。
  5. 備份套件內原本的 config.toml，把新檔放到 GamerCatch 資料夾最外層。Windows 要確認檔名沒有變成 config.toml.txt。
產生器永遠不需要讀取 JSON 內容 只填像 credentials/person-1-service-account.json 這種路徑文字。不要拖曳、貼上或上傳 JSON、private_key、client_secret 或 refresh token。
多人帳號不是多人登入網站 產生器只建立本機設定檔，不會登入 Google，也不會替您保存帳號。不同使用者的 Sheets 權限由各自的 service account JSON 與試算表分享設定決定。
13

認識產生器與 config.toml 欄位

欄位 小白版說明
enabled 開啟才會處理；暫時不用的遊戲保持關閉。
game_name 巴哈排行卡片上的完整名稱，空格與符號都要相同。
write_to_google_sheets 第一次保持 false；確認抓取正確後才開啟。
spreadsheet_id 直接貼完整 Google 試算表網址最簡單。
service_account_key_path 只填本機相對路徑，不貼 JSON 內容。
worksheet_name 試算表底部的分頁名稱，不是上方檔案名稱。
timezone 台灣通常使用 Asia/Taipei。
first_data_row 第 1 列是標題時填 2。
date / rank / popularity column 日期、排行、人氣欄，例如 A、B、C。
notification_recipients 空白時使用 Gmail 全域預設；填入時只通知該遊戲負責人。

若需要手動編輯 TOML

  • 只修改等號右邊；引號內的文字可以換。
  • true / false 必須小寫且不加引號。
  • 頁碼與列號是數字，不加引號。
  • 多人電子郵件要各自加雙引號並以逗號分開，例如 ["a@example.com", "b@example.com"]。
  • 不要刪除 [[games]]、欄位名稱、等號、雙引號或方括號。
  • 建議保留 headless = false，遇到巴哈安全驗證時才看得到瀏覽器。
  • 掃描頁數最多 200；頁數越多，執行時間與網站負擔越高。
14

三個遊戲怎麼設定

先完成遊戲 1。遊戲 2、3 使用相同欄位，但換成各自名稱、試算表、JSON 與收件人。

[[games]]
enabled = true
game_name = "夜鴉"
write_to_google_sheets = false
spreadsheet_id = "https://docs.google.com/spreadsheets/d/第一張ID/edit"
service_account_key_path = "credentials/person-1-service-account.json"
worksheet_name = "每日排名"
timezone = "Asia/Taipei"
first_data_row = 2
date_column = "A"
rank_column = "B"
popularity_column = "C"
notification_recipients = []
[[games]]
enabled = true
game_name = "請填巴哈上的第二個完整名稱"
write_to_google_sheets = false
spreadsheet_id = "https://docs.google.com/spreadsheets/d/第二張ID/edit"
service_account_key_path = "credentials/person-2-service-account.json"
worksheet_name = "每日排名"
notification_recipients = ["person-2@example.com"]
三張表都用 A / B / C 沒問題 每個遊戲寫到不同試算表時，欄位不會衝突。只有寫入同一張試算表、同一個分頁的不同遊戲，排行與人氣欄位才不能重疊。
15

第一次先做不寫入的安全測試

所有遊戲先保持 write_to_google_sheets = false 程式會抓排行與人氣，但不會修改任何試算表。強制 dry-run 發現錯誤時也不會寄 Gmail 通知。
  1. 儲存 config.toml，確認要測試的遊戲已啟用、寫入仍關閉。
  2. macOS 雙擊 2_開始抓取.command；Windows 雙擊 2_開始抓取.cmd。
  3. 瀏覽器開啟後等待；若出現巴哈安全驗證，請自行完成。程式不會繞過 CAPTCHA。
  4. 逐一核對遊戲名稱、排行、人氣與 page 是否和巴哈頁面相同。
  5. 看到「write_to_google_sheets=false，未寫入 Google Sheets」才是此階段的正確結果。
抓取成功：夜鴉｜排行 123｜人氣 456｜page=5
夜鴉：write_to_google_sheets=false，未寫入 Google Sheets。

客服可能會請您用命令列的 --dry-run，它會強制所有遊戲不寫入、不寄信，即使設定檔誤開啟寫入。一般使用者不需要自行操作。

16

開啟正式寫入與平常執行

建議一次只開一個遊戲，確認試算表真的寫對，再開下一個。

  1. 在產生器把遊戲 1 的「寫入 Google Sheets」開啟，重新下載並替換 config.toml；其他遊戲仍保持關閉寫入。
  2. 雙擊開始抓取，等待顯示「Google Sheets 更新完成」與列號。
  3. 回到試算表，確認今天那一列的排行與人氣正確，沒有寫到別人的表。
  4. 依序開啟遊戲 2、3。任何一人的 JSON 或 Sheets 失敗，不會阻止其他遊戲繼續。
平常只要雙擊開始抓取 看到「全部處理完成」即可關閉視窗。成功時不寄信；異常時才彙整通知。執行紀錄保存在同資料夾的 last-run.log。
不要一天重複執行很多次 程式會更新今天同一列而不是新增列，但仍請合理控制頻率，避免增加巴哈與 Google API 負擔。
17

Windows：每天 09:00 自動抓取

這是 Windows 選用功能。09:00 是電腦本機時間；試算表要找哪一天，仍由每個遊戲的 timezone 決定。

先手動成功，再安裝排程 先完成不寫入測試與正式寫入；啟用 Gmail 時，也要用同一個 Windows 帳號完成首次授權並收到測試信。安裝排程不會立刻抓取。
  1. 把 GamerCatch 資料夾放到「文件」等不會移動的固定位置。
  2. 雙擊 3_安裝每天早上9點自動抓取.cmd。
  3. 確認畫面顯示 Task 名稱與下次執行時間。重複安裝會更新同一工作，不會建立副本。
  4. 隔天查看 last-scheduled-run.log；一個 Task 會處理所有已啟用的遊戲。

執行條件與限制

  • 使用目前登入的 Windows 帳號與標準權限，不要求管理員，也不儲存 Windows 密碼。
  • 鎖定畫面仍可執行；登出或完全關機時不會執行。
  • 睡眠時會依硬體與電源設定嘗試喚醒；錯過 09:00 時，之後登入且網路可用會補跑。
  • headless = false 會開 Chromium；背景執行遇到 CAPTCHA 時仍需改回顯示瀏覽器後手動處理。
  • 排程不會開 Google 登入頁。Gmail 授權失效時，用同一 Windows 帳號重新執行首次授權。
  • 停用或刪除：開啟「工作排程器」，在程式庫找到 GamerCatch-Daily-0900 開頭項目。
搬動資料夾或換新版後要重裝 Task 記住完整路徑。只改同一位置的 config.toml 不必重裝；移動、改名或換到新版資料夾後，請在新位置再次雙擊安裝腳本。

更換遊戲時要修改與確認什麼

不需要重裝 GamerCatch。先備份目前可用的 config.toml，再於產生器修改要更換的遊戲。

第一步先關閉寫入 新遊戲保持 write_to_google_sheets = false。確認抓到正確遊戲前，不要寫入試算表。
要檢查的設定 更換時怎麼改
game_name 改成巴哈排行卡片上的完整新名稱，空格與符號都要一致。
end_page 新遊戲不在掃描範圍時逐步提高；這會套用到所有遊戲。
試算表與分頁 換表時修改 spreadsheet_id 與 worksheet_name。
JSON 換擁有者或帳號時改 service account 路徑，並把新表分享給該 JSON 的 client_email。
日期列與欄位 核對 first_data_row、日期、排行、人氣欄，並先建立今天日期。
通知收件人 核對專用收件人；留空才會使用 Gmail 全域預設。
啟用狀態 啟用新遊戲並停用不用的舊項目，避免同時執行。
  1. 先確認新表已分享給正確 client_email，而且今天日期列存在。
  2. 先做不寫入測試，核對遊戲名稱、排行、人氣與 page。
  3. 測試正確後，一次只開一個正式寫入，再核對表、分頁、日期列與欄位。
  4. 找不到遊戲時先核對名稱，再逐步提高 end_page，不要一開始就設到最大。

發行檔不完整 完整解壓縮整個資料夾，不要單獨移動腳本或主程式。
Chromium 安裝失敗 確認網路、代理與磁碟空間，再執行首次設定。
設定格式錯誤 用網頁產生器重新下載；手改時檢查雙引號、等號與 [[games]]。
找不到遊戲 名稱要與巴哈完全相同，必要時逐步提高 end_page。
找不到 JSON 核對 credentials 路徑、檔名與是否誤成 .json.json。
Sheets JSON 格式錯誤 OAuth client 或 API key 不能替代 service account JSON。
Google 403 確認 Sheets API 已啟用，且試算表以編輯者分享給正確 client_email。

輸出欄位重疊 同一試算表同一分頁的不同遊戲不可共用排行或人氣欄。

私鑰、OAuth 與帳號安全

懷疑 service account JSON 外洩

  1. 立刻到 Google Cloud 的 service account「金鑰」頁面，停用或刪除舊金鑰。
  2. 建立新 JSON，替換 credentials 裡的舊檔。
  3. 若新金鑰仍屬同一 service account，已分享的試算表通常不必重新分享。

懷疑 Gmail OAuth 外洩

  • OAuth JSON 含 client secret；到 Google Cloud 刪除舊 OAuth client，建立新 client 並重新授權。
  • 到 Google 帳號撤銷應用程式存取權，再重新執行 Gmail 首次授權。
  • refresh token 只應存在 Keychain 或 Credential Manager，不能出現在設定檔或問題回報。

求助時可以提供

  • 作業系統、x64 / arm64、GamerCatch Release 版本。
  • 完整錯誤畫面、last-run.log 或 last-scheduled-run.log；上傳前先檢查個人路徑。
  • 隱藏試算表網址與電子郵件後的 config.toml。
絕對不要提供 service account JSON、Gmail OAuth JSON、client_secret、refresh token、private_key，或可編輯的 Google Sheet 公開連結。
21

更新 GamerCatch

  1. 備份舊資料夾中的 config.toml 與整個 credentials。
  2. 從官方 Release 下載新版並核對 SHA-256，解壓縮到新的資料夾，不要直接覆蓋舊版。
  3. 用新版設定產生器重新產生設定，或把已確認相容的設定與 JSON 複製到新版。
  4. 先關閉所有 Google Sheets 寫入，重新做一次安全測試。
  5. Gmail 憑證屬於相同系統帳號時通常可沿用，但仍應執行首次授權確認測試信。
  6. Windows 每日排程使用舊路徑；在新版資料夾重新執行排程安裝腳本。
  7. 全部驗證完成後再保留或安全刪除舊資料夾。

下載後核對 SHA-256

雜湊像檔案指紋，可以確認下載檔沒有被破壞或換掉。先在同一個 Release 下載 SHA256SUMS.txt，再計算 ZIP。

macOS 終端機

shasum -a 256 GamerCatch-macOS-arm64.zip

Windows PowerShell

Get-FileHash .\GamerCatch-Windows-x64.zip -Algorithm SHA256
  1. 在 ZIP 所在資料夾開啟終端機或 PowerShell，執行對應指令。
  2. 把顯示的 64 個十六進位字元與 SHA256SUMS.txt 中相同檔名那一行逐字比較；大小寫可不同。
  3. 完全相同才解壓縮。只要一個字不同，就刪除 ZIP，回官方 Release 重新下載。
